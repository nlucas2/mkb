# Don't repeat — reuse

The whole point of mdkb is to write a thing **once** and reuse it everywhere.

1. Before adding something new, **search** for it.
2. If it exists, **embed** or **link** it instead of copying.
3. If a chunk inside a block is independently useful, **carve** it into its own block,
   replaced in place by an embed, so other blocks can reuse it too.

Edit a shared block once and every page that embeds it reflects the change — that is
transclusion, and it is how the docs in this repo stay free of duplicated facts.
