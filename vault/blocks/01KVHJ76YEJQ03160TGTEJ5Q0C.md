# Searching

Search runs a **hybrid** of keyword (full-text) and semantic (vector) ranking, so a
natural phrase works as well as exact terms. Search for a *concept*, not a single
keyword, and narrow by a `#tag` or by code language when useful.

Beyond text, search also filters on **metadata**: by recency (`created`/`updated` date ranges —
e.g. blocks not touched since a date) and by **property presence** (`has:` / `missing:` a property
key — e.g. atoms missing a `source`). Those power tidy-up audits across the whole vault.

Search before you write, so you reuse an existing block instead of duplicating it.
